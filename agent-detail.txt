# Jupiter AI Discovery Engine - Architecture Design Prompt

You are a Principal AI Architect designing the Discovery Engine for an AI Governance platform called Jupiter AI.

## Background

Jupiter AI is an AI Governance Auditor for enterprise AI systems.

Before auditing a customer's AI system, Jupiter must first understand the system in detail.

The goal is NOT to collect static form information.

The goal is to conduct an intelligent interview and build a Digital Twin of the customer's AI application.

The Discovery Engine should behave like a senior AI auditor who asks intelligent follow-up questions based on the customer's answers.

---

# Objective

Design a Discovery Engine that:

1. Understands the customer's AI system through conversation.
2. Extracts structured facts from the conversation.
3. Identifies missing information using a Discovery Schema.
4. Performs Gap Analysis continuously.
5. Dynamically asks the next best question.
6. Builds a complete System Profile.
7. Maps discovered entities to a Governance Knowledge Graph.
8. Produces a Digital Twin of the customer's AI system.

The Discovery Engine must maximize information gained while minimizing the number of questions asked.

---

# Important Design Principle

Do NOT use fixed questionnaires.

The Discovery Engine should:

* Extract multiple facts from a single answer.
* Identify unknown areas.
* Prioritize high-risk missing information.
* Ask adaptive follow-up questions.
* Stop asking questions once confidence is sufficient.

The system should feel like an intelligent conversation rather than a compliance form.

# Fact Extraction

* Extract facts from user messages.
* Extract multiple facts from a single answer.
* Assign confidence scores.
* Store source messages.
* Track contradictions.
* Detect assumptions.

Example:

"We built a banking support assistant using LangGraph and a CRM."

Extract:

Industry

Framework

Tool

System Type

Store provenance for every extracted fact.

# Gap Analysis Engine

Design a Gap Analysis Engine.

Requirements:

* Compare discovered facts against the Discovery Schema.
* Identify missing information.
* Prioritize gaps.
* Estimate discovery completeness.

The engine should know:

What is known?

What is missing?

What is most important to ask next?

Provide a scoring framework.

---

# Question Selection Engine

Design an adaptive Question Selection Engine.

Requirements:

* Ask the highest value question next.
* Avoid repetitive questions.
* Avoid already answered questions.
* Consider discovery completeness.
* Consider risk level.
* Consider system type.

Examples:

If RAG is detected:

Ask about documents, retrieval and vector stores.

If Agentic Architecture is detected:

Ask about tools, permissions and actions.

If Financial Advice is detected:

Ask about approvals and compliance controls.

Design the complete logic.
